# 🏗️ [手搓一切]：自顶向下手搓学习大模型

> **"如果知识你不能手搓出来，你就并不真正拥有它。"**

---

## 🚩 本项目的起因：这半年碰到了好多“调参侠”和“屎山代码”

这一年里我见了好多：
1. **盲人摸象式调参：** 对着超参表一顿乱改，却说不清这超参是干什么的，报错后不会改。
2. **逻辑崩坏式屎山：** 一个py文件写两三千行，又或者过度封装导致Debug时要反复横跳。
3. **GPT的人肉agent：** 问题、报错全扔给gpt，然后不管gpt说得对不对都照着搞。

**so我决定把大模型拆解开，带你用最原始、最纯粹的方式“搓”一遍。**

**原本只是想写出来给实验室当入门教程，后面一寻思，写都写了，直接放出来得了。**

---

## 🗺️ 进化路线图：从Attention到Model再到Training

在这里，我们不求模型效果（数据够小、模型够轻），我们只求 **“彻底跑通”** 和 **“逻辑通畅”**。

### 🧩 [Part 1：从 0 搓出一个 Model 类](./part1/)
* **核心：** 基于 Transformer 架构，实现自回归 Token 输出。
* **目标：** 让你明白一个“黑盒”是怎么从输入一个 ID 变成输出一串人话的。

### 👁️ Part 2：常见的Attention脚手架
* **核心：** 手搓 MHA, RoPE, MLA，online softmax，page attention等常见的东东
* **目标：** 明白现在的研究也好实现也好是在Part1中往哪些方向改变，这些改变就是可以研究的方向。

### 🎓 Part 3：通用的SFT流程
* **核心：** 基于`transformers`库，构建一套好看的、易改的sft方案。
* **目标：** 拒绝乱塞逻辑，我们要的是有组织、有纪律的sft代码套路。

### ⚖️ Part 4：通用的 RL（强化学习）对齐
* **核心：** 手搓RLHF/PPO/DPO等算法逻辑。
* **目标：** 搓完就知道他们都改了什么，以及还能怎么改了。

## 🧠 我的设计哲学 (The Rules)

* **🚫 拒绝公式迷恋：** 我们不用各种图展示公式，直接用代码实现，困惑时随时改数随时debug。
* **🚫 拒绝过度封装：** 杜绝两千行文件，杜绝过度套娃逻辑。
* **✅ 坚持需求驱动：** 所有的模块存在，都是为了解决特定的工程痛点，不是现在提出一个东西但是根本什么时候会用他。
* **✅ 坚持小而美：** 用最小的数据集甚至是随机tensor跑通最复杂的逻辑。
